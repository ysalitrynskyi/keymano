# Getting help with Keymano

Thanks for using Keymano. Pick the channel that matches what you need:

## Questions, ideas, layout-design help

Use **[GitHub Discussions](https://github.com/ysalitrynskyi/keymano/discussions)**.
Good fits: "how do I make a dead key for ã?", "is there a way to ...", "show off
the layout you built". Search first — someone may have already asked.

## Bug reports

Open a **[Bug report issue](https://github.com/ysalitrynskyi/keymano/issues/new?template=bug_report.yml)**.

Please include:

- what you did, what you expected, what happened instead,
- the Keymano version (About ▸ Version, or the GitHub Release tag),
- the platform (macOS / Windows / Linux / web build). **macOS won't open the app?**
  See [Getting Started — First launch on macOS](docs/GETTING_STARTED.md#first-launch-on-macos-important).
- if the bug involves a specific `.keylayout` / `.bundle`, attach a minimal
  reproducer — never send anything private.

If you can reproduce on the hosted web app at
[`keymano.ys.contact`](https://keymano.ys.contact), say so — it makes triage
much faster.

## Feature requests

Open a **[Feature request issue](https://github.com/ysalitrynskyi/keymano/issues/new?template=feature_request.yml)**.
Describe the underlying problem you're trying to solve, not only the proposed
solution. Describe the workflow you need and how you would use it.

## Security vulnerabilities

**Do not file a public issue.** Follow [SECURITY.md](SECURITY.md): use GitHub's
private vulnerability reporting or email **ysalitrynskyi@gmail.com**.

## Response expectations

Keymano is maintained by volunteers. Expect best-effort responses; security
reports are prioritized. There is no paid support tier.
